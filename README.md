# Olist EC事業 リピート率改善分析プロジェクト

ブラジル最大のECサイト「Olist」のデータを用い、リピート率向上のためのボトルネックを特定しました。

## プロジェクト概要
売上成長の鈍化を背景に、顧客の継続利用（リピート）に着目。
SQLによるデータ抽出とLooker Studioによる可視化を組み合わせ、経営判断に直結するダッシュボードを作成しました。

## ダッシュボード構成
### 1. 経営KPIサマリー
月次の売上、顧客数、およびリピート率の推移をモニタリング。
- **技術的工夫:** Window関数(`ROW_NUMBER`)を用いて、全期間におけるユーザーごとの購入順序を厳密に判定し、リピート率を算出。

### 2. リピート要因分析（配送日数）
配送の早さがリピート率に与える影響を分析。
- **分析結果:** 配送が21日を超えるとリピート率が最小（2.88%）となることを特定。
- **提言:** 全体の約14%を占める「遅延層」の解消に向けた物流網の再考を提示。

## 使用技術
- **データ基盤:** Google BigQuery (SQL)
- **可視化:** Looker Studio
- **分析言語:** Python (要因探索フェーズで使用)

## 構成
- `/sql`: 分析に使用したクエリ
- `/images`: ダッシュボードのスクリーンショット
